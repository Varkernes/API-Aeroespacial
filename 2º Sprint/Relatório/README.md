## - Segunda Sprint

 Esta etapa tem seu início no mês de outubro após a apresentação da primeira sprint, onde teve início à uma coleta de dados filtradas no comexstat em busca do valor FOB, nas importações e exportações no segmento aeroespacial da região metropolitana do vale do paraíba e litoral norte.

## 1. Desenvolvimento – Segunda Sprint


Para avaliação desta segunda parte do projeto, houve-se requisitos para serem atendidos, estes são: o uso do github; planilhas com dados extraídos no comexstat; o uso do PowerPlataform (Powerbi e Powerapps). Os itens citados, auxiliam para a visualização e demonstração dos dados tratados e também do processo para a conclusão do projeto.
O comexstat foi onde teve o início do processo de desenvolvimento do projeto, o banco de dados incluídos dentro deste site, podem ser filtrados por região e munícipios, uma vez que filtrados estes dados eles podem ser baixados em planilhas do excell, isso é fundamental para o projeto porque, essas filtragens e planilhas serão usadas diretamente no software do Power Bi. Uma vez que os arquivos retirados do site e abertos dentro do excell, são transformados em tabelas usados no Power Bi pois, o objetivo é transformar estes dados, em dados visíveis como por exemplo: visualização em gráficos; tabelas; indicadores; e também com os dados filtrados por região é possível a visualização dos dados através de um mapa interativo, este que é incluído no arquivo e no requisito do projeto.
Após a formatação dos arquivos no software Bi, é então transferidos para o Powerapps, onde é possível conectar o todo do projeto, ou seja, planilhas e os dados e também toda a construção no PowerBi, é demonstrada de uma forma ágil e unificada. O processo de desenvolvimento desta segunda sprint e todo o progresso e arquivos contidos neste trabalho em andamento, são postados e documentados no Github.

## 2. Pós-Sprint 


A avaliação do cliente em relação ao projeto apresentado, consta que para esta segunda sprint foi insuficiente visto que, a página no github apresentava uma escassez de detalhes e arquivos, dada quase como uma parte abandonada do projeto. Foi relatado que o Power Bi e Powerapps contia dados ainda fragmentados. Para a solução desta problemática foi explicado pelo cliente que, na página de base de dados dentro do site do comexstat haveria as planilhas de importação e exportação pedidas pelo cliente, também há a planilha com o código dos munícipios codificados, através dessas informações o grupo então, teria como objetivo para a terceira sprint, corrigir sua planilha, isto é, caracterizar e desfragmentar os dados e integrar esses novos códigos adquiridos na base de dados dentro do Powerbi e Powerapps.
